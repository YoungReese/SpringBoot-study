背景：当数据写入不是后端写入，而是通过第三方数据端同步写入mysql，写入数据先清除数据，然后同步，因为数据量很大，需要40-60秒完成，
     前端此时请求后端数据，经常会出现要等待很久的情况，等待时，前端web界面没有数据显示！

解决方法：由于这种信息展示，对数据的实时性不要求100%精确，所以考虑使用缓存（当我实现完demo，写此文档时，发现可以直接使用map），
        每次把缓存数据返回出去，然后开启一个异步任务更新缓存，用于下一次展示，异步任务中，查询的数据为空（数据库正在更新数据），
        不更新redis缓存，直到下一次有数据才更新，因此，总有数据返回给前端展示，保证用户体验。
